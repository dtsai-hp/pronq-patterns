Mockups and Designs

When browsing through the Online Business Team DropBox or building new designs, use the following folder structure and file naming conventions to find/save files.

Product Naming

Full Product Name Abbrevation Solution
Agile Manager agm Project Tracking & Collaboration
Application Defender hpad Security
AppPulse Active appactive Application Performance Management
AppPulse Mobile appmobile Application Performance Management
AppPulse Diagnostic appdiag Application Performance Management

StormRunner Load srl Load Testing

Vertica on Demand vod Big Data & Analytics
Virtualization Performance Viewer vpv Application Performance Management

Folder Structure

Project (i.e. Product_Details, Global_Assets)
  • explorations
  • global
  • product name
    • wireframes
    • mockups
    • research
    • content (all original assets and content from PMs)
    • prototypes
    • visual design
      • assets
      • source
      • mockups
Spacing Use spaces in between words, no underscores (_) or dashes(-)
Casing Use lowercase only
Product Names Abbreviate product names as much as possible

File Naming

*(product_) + *(project_) + (platform_) + (version_) + (designer_) + (finaldetail) . extension
Platform Specifications Dimension Document Width Content Width

Landscape Tablet landscapetab 1200px 970px
Portrait Tablet portraittab 990px 750px
Mobile mobile 320px 320px
Spacing Only use underscores (_), no dashes (-)
Casing Use lowercase only
Designer Names Use a designers initials if including a name
Product Names Abbreviate product names as much as possible
Version # - Version/concept, letter - Iteration
Final details ie. Specs, screenshots
